## 04. Testing

Damit Python deine Module innerhalb von `src/` findet, musst du das aktuelle Verzeichnis in den `PYTHONPATH` aufnehmen. Wechsle in dein Projekt-Root-Verzeichnis und führe aus:

### Unter Linux / macOS / Git Bash:

```bash
export PYTHONPATH="${PYTHONPATH}:$(pwd)"
pytest
```

### Unter Windows (PowerShell)

```powershell
$env:PYTHONPATH = ".;$env:PYTHONPATH"
pytest
```

### Die Konfiguration automatisieren

Damit du nicht jedes Mal den Pfad manuell setzen musst, kannst du eine Datei namens `pytest.ini` oder einen Abschnitt in deiner `pyproject.toml` im Root-Verzeichnis anlegen:

```toml
[tool.pytest.ini_options]
pythonpath = ["."]
testpaths = ["tests"]
asyncio_mode = "auto"
```

### Tests mit Codecoverage

Um die Coverage für dein Projekt zu messen, musst du beim Starten von `pytest` angeben, welches Verzeichnis (dein Quellcode) überwacht werden soll.

```bash
pytest --cov=src/org/slashlib/py/agent tests/
```

Oder in der `pyproject.toml` einfügen

```toml
[tool.pytest.ini_options]
addopts = "--cov=src/org/slashlib/py/agent --cov-report=term-missing"

[tool.coverage.run]
source = ["src/org/slashlib/py/agent"]

[tool.coverage.report]
show_missing = true

[tool.pytest.ini_options]
addopts = "--cov --cov-report=xml --cov-report=html"
```
