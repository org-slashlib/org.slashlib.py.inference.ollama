## 07. OllamaInferenceAdapter

**logger category:**  `org.slashlib.py.inference.ollama.adapter`

---
### async OllamaInferenceAdapter::chat(...) -> agent.InferenceResult

#### Parameters:
- **model:**  optional String, defaults to value found in `pyproject.json`
- **messages:** optional List of Dictionaries, defaults to None
- **tools:** optional List of Objects returned by @org.slashlib.py.agent.tool, defaults to None
- **kwargs:**
	- **think:** True | False
	- **timeout:** int (seconds), defaults to value found in `pyproject.json`

#### Returns
- **agent.InferenceResult**

#### pyproject.json

Default Values für `model`, `think` und `timeout` können über die `pyproject.json` festgelegt werden

```json
{
  "inference": {
    "ollama": {
      "model":      "gemma4",
      "think":      true,
      "timeout":    600.0
    }
  }
}
```

#### Thinking Mode

If a model supports reasoning ("thinking"), this feature can be controlled by passing the boolean parameter `think = True | False`. 

While the thinking process delivers higher-quality results (especially when combined with tool usage), it consumes more tokens and increases response time. Therefore, it can be beneficial to disable thinking if no tools are specified for a call. 

To handle this efficiently, the value of the `think` parameter is evaluated through a **three-tier decision logic**:

1. **Explicit Argument:** Thinking is explicitly enabled or disabled if the `think` keyword argument is passed as `True` or `False`.
2. **Implicit Tool-Based Fallback:** If `think` is not specified, it automatically defaults to `True` if the `tools` parameter is provided and not empty.
3. **Configuration Default:** If no tools are specified (or the parameter is `None`), the `think` value defaults to the setting defined in `pyproject.json`.
